# 数据流与刷新说明

> 目录结构详见 [目录结构说明.md](./目录结构说明.md)

## 1. 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│  main.dart                                                        │
│  初始化 DioClient → ProviderScope.overrides(dioClientProvider)     │
└───────────────────────────────┬─────────────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────────────┐
│  UI (presentation)                                               │
│  ConsumerWidget + ref.watch(homeListProvider)                    │
│  下拉刷新 → ref.invalidate(homeListProvider)                      │
└───────────────────────────────┬─────────────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────────────┐
│  Provider (providers)                                            │
│  FutureProvider.autoDispose<List<HomeItem>>                      │
│  刷新：ref.invalidate / ref.refresh                              │
└───────────────────────────────┬─────────────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────────────┐
│  Repository (data)                                               │
│  HomeRepository(DioClient) + ApiPath → List<HomeItem>            │
└───────────────────────────────┬─────────────────────────────────┘
                                │
┌───────────────────────────────▼─────────────────────────────────┐
│  DioClient (network)                                             │
│  get/post/postPb 等基础 HTTP 请求                                │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 分包职责

| 包 | 职责 |
|----|------|
| **main** | 初始化 DioClient、ProviderScope.overrides 注入 |
| **core** | ApiPath 路径常量、dioClientProvider 定义、跨 feature 共享数据 |
| **network** | DioClient 基础 HTTP 请求（get/post/postPb） |
| **feature_xxx/data** | Model、Repository（DioClient + ApiPath） |
| **feature_xxx/providers** | Riverpod Provider（依赖 dioClientProvider） |
| **feature_xxx/presentation** | UI、ref.watch、下拉刷新 |

## 2.1 分包数据隔离（重要）

- **禁止** feature 包之间互相调用数据（不 import、不 watch 其他 feature 的 Provider）
- **每个 package 只提供自己的公共数据**：通过导出 Provider 暴露本模块数据
- **跨 feature 共享数据**：放在 core 或 shared 包
- 各 feature 独立完成「UI → Provider → Repository → Network」闭环

## 3. 刷新方式

### 3.1 下拉刷新

```dart
RefreshIndicator(
  onRefresh: () async {
    ref.invalidate(homeListProvider);  // 使 Provider 失效，自动重新请求
  },
  child: ...,
)
```

### 3.2 按钮重试

```dart
FilledButton(
  onPressed: () => ref.invalidate(homeListProvider),
  child: const Text('重试'),
)
```

### 3.3 代码中主动刷新

```dart
ref.invalidate(homeListProvider);   // 失效并等待下次 watch 时重建
ref.refresh(homeListProvider);      // 立即重新请求并返回新值
```

## 4. 扩展其他 feature

其他 feature（如 feature_detail）可复用相同模式：

1. 在 `data/` 下添加 model、repository
2. 在 `providers/` 下添加 Provider（可用 `family` 传 id）
3. 在 presentation 中用 `ConsumerWidget` + `ref.watch`

## 5. 初始化与 baseUrl

在 `main.dart` 中初始化 DioClient 并注入：

```dart
void main() {
  final dioClient = DioClient(
    baseUrl: 'https://your-api.com',  // 替换为实际后端地址
    connectTimeout: const Duration(seconds: 10),
    receiveTimeout: const Duration(seconds: 10),
  );

  runApp(
    ProviderScope(
      overrides: [dioClientProvider.overrideWithValue(dioClient)],
      child: MyApp(router: router),
    ),
  );
}
```

## 6. API 路径集中管理

所有 API 路径定义在 `core/lib/api/api_path.dart`：

```dart
class ApiPath {
  static const String homeList = '/api/home/list';
  static const String homeDetail = '/api/home/detail';
  // 新增路径在此追加
}
```

禁止在 feature 包内散落路径字符串。
